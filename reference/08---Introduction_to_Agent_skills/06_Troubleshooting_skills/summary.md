# [Troubleshooting skills](https://anthropic.skilljar.com/introduction-to-agent-skills/434530)

## Summary

- トラブルシューティングの方法をまとめている章である
  - *Validator tool*を使えるようになるべし
    - インストール方法の一例 : `uv sync`
    - 実行すると構造的な問題がある場合、どこからでもキャッチしてくれる。
    - まずはこれを実行してから次に進もう。
  - *Skiils*がトリガーに引っかからない場合の対処方法
    - `description`は適切でない可能性がある。
    - 実際にユーザーが効きそうな言い回しでプロンプトを投げトリガーするかどうかをテストし、トリガーしなければその文言を`description`に追記するのも一つの方法だ。

  - *Skills*がロードされない場合の対処方法
    - ディレクトリ構成は正しくない可能性がある。
    - `.claude/skills/{SKILL_NAME}/SKILL.md`となっており、frontmatterのnameとディレクトリ名が一致しているかを確認しよう。
    - ターミナルで`claude --debug`を実行することでSkillの読み込みに関するエラーが発生していないかを確認するのも良い方法だ。
  - 想定と異なる*skills*がロードされてしまった際の対処方法
    - `description`が似すぎている可能性が高い。もっとSkillの使用条件などを明確にして差別化するべし。
    - description が似ているケースとは別に、**同名のスキルが enterprise / personal / project / plugin の優先度で上位に上書きされ、下位のスキル（例: 自分の personal スキル）が無視される**問題もある。対処は「自分のスキルをより固有の名前にリネームする（通常こちらが簡単）」か「管理者に enterprise スキルの相談をする」の2択。
  - *runtime error*が発生した場合の対処方法
    - 外部のライブラリなどを使用するSkillなどの場合はインストールされていることやそのライブラリとの依存関係がちゃんと明記されているかを確認すること
    - Claudeが実行するための許可を設定すること(`chmod +x`)
    - PATHの区切り文字(`/`であること)を確認しよう(Windowsでも`/`. (Backslash:`\`じゃないということ))

### Note/Tips


## Supplement

- プラグインのスキルが表示されない場合の対処: *キャッシュをクリア* → *Claude Code を再起動* → *再インストール*。それでも出ないならプラグインの構造自体が誤っている可能性が高く、Validator が役立つ。

- 6カテゴリの早見表:
  1. *発火しない* → description・トリガー語に不備がないか確認すべし
  2. *ロードされない* → パス/ファイル名/YAML syntaxを確認すべし
  3. *別スキルが使われる* → description を差別化
  4. *シャドウされる* → 優先度階層を確認し必要に応じてリネームすべし
  5. *プラグインが出ない* → キャッシュ削除 + 再インストールすべし
  6. *実行時エラー* → 依存関係/権限/パス区切りが正確か確認すべし
