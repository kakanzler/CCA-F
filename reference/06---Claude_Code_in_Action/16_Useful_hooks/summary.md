# [Useful hooks!](https://anthropic.skilljar.com/claude-code-in-action/312004)

## Summary

- この章ではhooksが有効となるケースを列挙している。
  1. 型チェック
     - Claudeは何かの変数や関数なら引数の数、引数の型などを変更したりすることがあるが、関数の呼び出し元まで変更しないケースがあるため、*PostToolUse*をつかって、変更後に型チェックツールを実行し、エラーがあればそのままClaudeにわたし、エラーを直させるというのが有効。なお、この方法は型ツールがある言語はもちろん、Pythonなどの言語であっても、同様のテストを実装・実施させることで代替可能
  2. 既存の関数などのリソースを再利用するようなレビュープロセス
     - Claudeは既存の関数があるのにもかかわらず同じようなものを0から作るときがある。この時レビュープロセスが有効となる。
       - レビュープロセスの仕組み
         - Claudeを二つのインスタンスにプログラム上分ける
         - 一つのインスタンスではコードを修正・作成し、もう一つのインスタンスで、作成されたコードと既存コードを比較し、再利用できるコードはないかをチェックさせる。もしあれば、Claudeにその旨を伝え、修正させる。

- より一般化すると
  - コンパイラやLinterの出力をClaudeへFeedbackとして入力
  - AIを分割し、コード作成とコードレビューで役割を分けるプロセスを作る
  - 最も重要なディレクトリなどに絞って実装する
  - 自動化によるメリットとコストなどについて費用対効果を考察する

- 最も大切なことは自身のプロジェクトにおいて課題となる部分はどこか特定し、その対処についてHooksによる自動化のターゲットを見極めること

### Note/Tips

- Summaryに書いた2つの手法のうち、1つ目は処理として軽いが2つ目は2つに分けたりなど重い処理となる。以下のトレードオフを考えたうえで利用するのがよく、重要な変更などに絞って利用することが推奨されている。
  - 利益：重複の少ないきれいなコードベースを作成可能
  - コスト：API使用料多い
  - 時間：長い

## Supplement

‐ 2つ目のClaudeが2つのインタンスに分けるような動作はSDKを使用して構成される。このようにプログラムてきにClaudeのAgent SDKを利用することで高度なフローを実現可能
