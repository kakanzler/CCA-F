# [Controlling context](https://anthropic.skilljar.com/claude-code-in-action/303237)

この章での目的はClaudeが話の流れを剃らないようにコントロールするテクニックの紹介。Claudeを生産的にするための必須の技術だ

## Summary

- `Escape Key` : 現在の推論を途中で抜ける際のコマンド
  - 有効なケース
    - Claudeが複数のことを同時にしようとしている時、Escape Keyを押下し作業を一時中断させ、「一つのことに絞って推論して」のようなPromptを投げて集中させるときに有効。
    - 同じミスを繰り返している時、Escape Keyで抜け、　`/memory` or `CLAUDE.md`を直接編集により正しいアプローチなどを記載し、再度推論させる

- `/rewind`(or 2回 Escape key押下 )：会話を巻き戻すコマンド。押下するとメッセージ履歴が表示されるので巻き戻す位置を任意に選択する。
  - 会話が長くなりすぎるとContextが蓄積し、無関係な情報によってClaudeが気を散らす原因になりうる。その際に会話を巻き戻すことが有効なケースがある。
  - 価値あるContext（コードベースの理解など）を保持したまま、邪魔な履歴だけを除いて現在のタスクに集中させたいときに有効
- `/compact` : ClaudeがPromptのやり取りの中で学習した重要な情報を保持しながら、会話履歴を要約するコマンド。
  - 会話が長くなりすぎたが、Claudeは重要な情報にたどり着いているため作業を続けていきたいそんな際に有効。
- `/clear` : 全く異なる話題にしたいとき、あるいは、それまでの会話の内容が新しく始めたい会話の妨げになりうるとき、FreshなContextで会話を始められるようにするコマンド



### Note/Tips

- `/clear` によって新しいContextで会話を始められるが、前の会話に戻りたいとき、`/resume`によって、戻ることができる。 `/clear`はセッションの会話履歴を削除するコマンドではないことに注意。
