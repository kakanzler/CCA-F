# [Making changes](https://anthropic.skilljar.com/claude-code-in-action/303236)

## Summary

- **核心：Planning と Effort は別軸の複雑さに効く**
  - Planning Mode（広さ／Breadth）：コードベース全体の理解・多段階・複数ファイルにまたがる変更向け。実装前に多くのファイルを読み、計画を立てて承認を待つ。
  - Effort level（深さ／Depth）：複雑なロジック・デバッグ・アルゴリズムなど、1点を深く詰める問題向け。

    - `/plan` : プランモードへの切り替えコマンド。 `Shift + tab`で切り替える操作と同値
    - 得意(Breadth／広さ)：
        - コードベース全体を理解する必要があると
        - 多段階的な推論が必要になるとき
        - 複数のファイルを修正する必要があるとき

    - `/effort`：推論のレベルを確認し、切り替えるコマンド
    - `low` : 速いが、質はそこまで高くない。
    - `max` : 質は高いが長い時間を要するので、複雑な問題などに対して使用すると効果的。
    - 得意(Depth／深さ)
        - 複雑な論理構造を持つ問題を解くとき
        - 難しいデバッグをする際
        - アルゴリズム的な問題を解くとき

### Note/Tips

- Claudeとの正確なコミュニケーションのためにScreenshotsを使うこと推奨
  - 貼り付けはmacOSでも`Cmd+V`ではなく`Ctrl+V`（非自明な点）
- プランモードにおいてClaudeのReviewが意図にそぐわない場合、`Ctrl + G`を使ってテキストエディタを開き、指示をより正確に修正し、Claudeに再度計画を練らせることができる。
- Claudeが推論中、「推論しています」などの表示はでるが、より詳細に情報が知りたい場合は、`Ctrl + O`を使うことで、より実際の推論のステップを表示できる
- `ultrathink` : このPromptに関してはより高度に推論してくれ、1ターンだけの限定的な指示をしたい場合にPromptに含める文字列。

- プランモード（広さ） ＋ /effort（深さ） のように両方とも有効にして推論することも可能だが、その分Tokenを消費するということに注意
