# Other ways to work with Claude

## Summary

- AIM : Claudeは知能そのものであり、各ツールはその知能を作業現場に届ける"surface"だと理解し、用途に応じてどのsurfaceを使うべきかを判断できるようになること。

- Claude Code
  - コード、コマンド、自然言語で記されたワークフローを理解させたいとき
  - 自然言語で用件を伝え、Claudeにコードを書かせ、テストさせ、コミットさせたいとき
  - Claudeにエラー解析させ、修正させたいとき
  - 慣れ親しまないコードベースをりかいしたいとき
  - Lint Error、MergeConflictなどの解消させたいとき
  - 自分のTerminal/IDEに加え、browserやSlack上でもコードベースを理解させ、コマンド実行〜workflow全体を任せたいとき


- Claude in Slack
  - threadの要約や、メッセージへの回答のレビューをSlack内で完結させたいとき
  - 新しいチームに入ったときに会話履歴から素早くオンボーディングするとき
  - Coding Taskを`@claude`のメンションでClaudeCodeになげたいとき

- Claude for Excel
  - 複数のシートをまたがるような複雑な計算式の入ったエクセルシートを利用する場合
  - 依存関係などが破綻しないように部分的にUpdateするひつようがあるとき
  - チャート、グラフ、テーブルなどを作成したいとき

- Claude for PowerPoint
  - ノートやドキュメント、概要からスライドを一通り作成したいとき
  - 話し言葉に変換してスピーカーノートに書き写したいとき
  - スライドを並び替えたり、スライドの構成を修正したいとき
  - 一貫性のあるLayoutに修正したいとき(ブランドのスタイルはそのままに)
  - チャートなどのVisualizingを適切に配置したいとき

- Claude for Chrome
  - 記事の要約、論文の調査などをブラウジングせずにしたいとき
  - メールの草案作成、メールボックスの管理したいとき
  - Webサイトの機能テスト(E2E)などを自動で実行したいとき
  - ニッチなツールであってもタブ間の情報(Webサービス間)を統合的に解釈したいとき

### Note/Tips

- 表内の「Claude Cowork」は本文のセクション説明には登場せず、サマリ表でのみ言及される（複数ステップの複雑なタスク=リサーチbrief/資料作成/ファイル整理/データ分析向け、Desktop中心。モバイルはDispatch経由）。surfaceの一覧として押さえておくとよい。
