# Research mode for deep dives

## Summary

- Researchとは、WebやConnectorでつないだ情報などを多岐にわたる情報を用いて多角的な視点で完璧な回答・理解を得る機能。
> 欠落していた要素: Researchの価値の核である「本来何時間もかかる調査を数分で完了する（多くは5〜15分、複雑なものでも最大45分）」という所要時間の前提が抜けていました。
> 修正案: Researchとは、本来何時間もかかる調査を数分（多くは5〜15分、複雑なものは最大45分）で完了し、多角的な視点で完璧な回答・理解を得る機能。
  - 複数のレポートを調査し、統合した回答を得るのに時間がかかるケース、深く分析するケース、複数のCompetitorを様々なオプションを鑑みて比較分析するケース、正しいことを保証できる引用を持つ報告書が必要なケース
> 冗長な要素: 用途の列挙(深く分析する/比較分析する/引用付き報告書が必要…)が網羅的で冗長でした。代表例1つに絞れば足ります。
> 修正案: 複数の情報源を統合した包括的・比較的なレポートが必要で、引用で検証可能な回答が欲しいケース（例: Competitorの比較分析）。

- Researchを有効にするとすでに見つけた情報をもとに次何を検索すべきかを判断し多段階に検索するエージェントが有効になる
 1. Claudeはアプローチの計画を立てる(＝Extented Thinkingは自動で有効になり、ユーザーの質問を分解し、真に何が必要なのか、どう調べればよいかを多角的に考え、特定する)
 2. 多段階検索を実行する（＝目標までのギャップを埋めるための見込みを立て、次に何を探すかを決定する。）
 3. 得られた情報を統合する(＝GmaiやGoogle Driveなどに統合して出力する)
> 欠落していた要素: 「統合(synthesize)」を「Gmail/Google Driveに出力する」と捉えていましたが、実際はWebや連携先(Gmail/Google Drive等)を含む複数の情報源から"集めた"情報を、整理された包括的なレポートに"まとめる"工程である、という理解が抜けていました（情報の出力先ではなく入力元）。
> 修正案: 3. 得られた情報を統合する(＝Web・連携先(Gmail/Google Drive等)を含む複数の情報源から集めた情報を、整理された包括的なレポートにまとめる)
 4. 統合した情報に含まれる内容について、それぞれの引用元を記す 

### Note/Tips

- 逆に、1，2個の情報源からQuickに回答が欲しい際はResearchではなく、普通のWeb検索するほうが良い。思慮深さよりも迅速さが優先されるケースもResearchよりWeb検索。
- 外部リソースを必要としない複雑なケース(数学やデバッグなど)への思考などであればResearchではなく、extended thinking モードを利用したAIとの対話が適している
- 社内の情報(slack, メール、議事録など）に関する検索が必要な差異はClaudeの企業向けプランに含まれるEnterprise Searchが望ましい
- Research時のPromptのTips
  - 目的や必要情報、制約などを具体的に記載すること
  - よりよいPromptにするためにはどうすればいいかさえもClaudeにきいてしまってもよい
- Promptに明示する(「Slackやメール内に、～」、)ことでWebSearchを使わずにResearchを使うことも可能
- 情報を明示的に統合させる(「GoogleDriveから～」「最近のメールから情報とって～」)ことで意図通りの結果が得られる

## Supplement（Claude Code CLIでのSkills）

- Researchを有効にすると、Web search有効になる。この状態でResearchを無効にしても、Web Searchが有効のままになるので乗田管理には注意
