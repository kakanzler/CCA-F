# refine-summary

Anthropic Academyのまとめ作業用コマンド。現在開いているsummary.mdをorigin.mdと照合してrefineし、理解度を評価する。

## Steps

1. IDEで開いているファイルのディレクトリを特定する。開いているファイルがsummary.mdでない場合はその旨を伝えて終了する。

2. 同じディレクトリにあるorigin.mdを読む。

3. summary.mdを読む。

4. origin.mdと比較して、summary.mdをrefineする。**要約とはorigin.mdの転記ではない。**summary.mdに記載された元の文章は修正せず、フォーマットに従い追記する。

   **まず、概念的な新規性が薄い回は2段階で扱う（該当しなければ通常どおり下記の基準でrefineする）:**
   - **最小要約**（実演・デモ・Example 中心で、概念の核が1点は残せる回）: フル要約にせず核1点だけ書き「詳細は実演ベースなので origin 参照」とする。READMEは通常どおり **✅ 完了**。
   - **完全割愛**（セットアップ・インストール等、概念がほぼゼロの回）: summary.mdは割愛する旨だけ残し、refineは行わない。READMEは **➖ 割愛**（完了日は処理日）。

   通常のrefineは以下の基準で判断すること:

   **含めるべきもの（核心）**
   - 概念の定義・本質（「何であるか」「なぜ重要か」）
   - 他の概念との区別・対比（例: SkillsとProjectsの違い）
   - 読者が後から忘れやすい非自明な制約や前提。欠落の扱いは概念的な誤解・欠陥かどうかで分岐する:
     - **概念的な誤解・欠陥がある場合**（読者が間違った理解を示していた、または概念の核が抜けている）: 該当の文の次の行に以下のフォーマットで注意を追記する
       ```markdown
       元の文
       > 欠落していた要素: {{欠落内容}}が抜けていました。
       > 修正案: {{元の文章の修正案}}
       ```
     - **概念的な誤解・欠陥ではない場合**（核は理解できており、あれば役立つ補足的な情報）: 上記フォーマットは使わず、`## Supplement` セクションに箇条書きで追記する。Summary本文には手を入れない。

   **削るべきもの（冗長）**
   - 列挙された例のほとんど → 最も代表的な1つだけ残せば十分
   - 手順のステップ詳細 → 「〇〇から設定する」程度の一言で足りる
   - 「〜を学びます」「次のレッスンでは〜」などの案内文
   - origin.mdで繰り返されている同じ概念の言い換え

   **判断基準**: 「この一文がなければ概念の理解が変わるか？」で取捨選択する。変わらないなら削る。

   修正後の記載は該当の文の次の行に以下のフォーマットで記載する
     ```markdown
     元の文
     > 冗長な要素: {{冗長箇所}}が不要でした。
     > 修正案: {{元の文章の修正案}}
     ```

5. refineしたsummary.mdを保存する。

6. 元のsummary.md（refine前）の内容を基に理解度を評価する:
   - 総合評価（A/B/Cなど）
   - 良かった点（何を正しく理解できていたか）
   - 抜けていた点（何が見落とされていたか、それが概念的な誤解か細部の見落としかも示す）
   - 次回へのアドバイス

7. README.mdの学習進捗を更新し、コミットする:

   **ディレクトリパスの解析**
   summary.mdのパスから以下を抽出する:
   - `reference/XX---aaaaa/YY_bbbbb/summary.md` という構成を前提とする
   - `XX` = コース番号（例: `01`）
   - `aaaaa` = コース名（例: `Cluade_101`）
   - `YY` = モジュール番号（例: `08`）
   - `bbbbb` = モジュール名（例: `Connecting your tools`）

   **README.mdの更新**
   リポジトリルートのREADME.mdを開き、`## 学習進捗` 内の、コースディレクトリ `reference/XX---aaaaa/` へリンクした見出し `### XX — [aaaaa](reference/XX---aaaaa/)` に対応するセクションを探す。
   - モジュール名はsummary.mdへのリンク `[bbbbb](reference/XX---aaaaa/YY_bbbbb/summary.md)` とする。パスにスペースが含まれる場合は `%20` でエスケープする。
   - 完了日は今日の日付を `YYYY/MM/DD` 形式で記載する。
   - セクションが存在する場合: 既存の表に下記フォーマットの行を追記する。ただし、すでに該当の行がある場合はその行を修正する。
   - セクションが存在しない場合: `### XX — [aaaaa](reference/XX---aaaaa/)` の見出しと表を新規作成して追記する

   表のフォーマット（ステータスは Step 4 の判断に従う。通常・最小要約は `✅ 完了`、完全割愛は `➖ 割愛`）:
   ```
   | # | モジュール | ステータス | 完了日|
   |---|---|---|---|
   | YY | [bbbbb](reference/XX---aaaaa/YY_bbbbb/summary.md) | ✅ 完了 | YYYY/MM/DD |
   ```

   **コミット**
   README.mdをステージして以下の形式でコミットする:
   ```
   update : add lesson YY to progress table in README
   ```
